# Episode Blueprint — 270-Second Skeleton (4–5 min action episode)

Copy this template into `episodes/E<nn>-<slug>/BLUEPRINT.md` and fill every field. The structure
loop (qa-loops.md §Loop 1) checks against the numbers below — they are targets with tolerance,
not vibes. **Runtime definition:** runtime = story length before the button, target 270 s (4:30);
the button (≤5 s) and any title allowance sit outside it. All percentages below are of story
runtime.

## 1. Premise block

- **Logline (≤2 sentences):**
- **FORP engine check:** how does this premise express *fragments of real people* +
  *containment over destruction*? (Removal test: without the FORP title, does it still belong?)
- **Fragment:** which real person's fragment, and what truth of theirs does the episode dramatize?
- **Dramatic question:** the question the episode itself answers on screen.
- **Final 3 seconds (write FIRST):** the exact closing image.
- **Button (≤5s):** thematic sting or universe tag — NEVER a plot cliffhanger.

## 2. Compression gates (all must hold at lock)

- [ ] One moment in time (hours at most — never days)
- [ ] 1–3 speaking characters (silent extras unlimited)
- [ ] ONE core conflict
- [ ] 1–2 environments (a vehicle between them counts as a corridor, not a third)
- [ ] Dialogue minimal: every line either advances plot or reveals character under pressure
- [ ] Character-depth test: protagonist's want / pressure / visible change stated; if a different
      character could star in the same ending, the premise fails
- [ ] Fantastical element: capability AND cost defined per world `[PHYSICS]`

## 3. Beat skeleton (20/60/20)

| Clock | Beat | Function | Required content |
|---|---|---|---|
| 0:00–0:05 | **Detonation frame** | hook, zero context | strongest single image of the episode; mid-crisis |
| 0:05–0:15 | Hook completion | orient inside the crisis | protagonist visible + physical stakes legible |
| 0:15–0:45 | Escalation 1 | first fight/chase engagement | fantastical element's capability shown (not explained) |
| 0:45–1:00 | **Inciting turn / Act 1 out** | re-price the situation | mini unresolved beat at ~1:00 boundary |
| 1:00–2:10 | Act 2a: beats 1–2 | tug-of-war | each beat ends in a STATE-CHANGE; jolt every 30–45s |
| 2:10–2:20 | **Midpoint spike** | biggest re-pricing before climax | power reversal #1 (prop/environment-triggered) |
| 2:20–3:40 | Act 2b: beats 3–4 | cost mounts | element's COST shown; mini unresolved beat ~2:15 & ~3:40 |
| 3:40–4:05 | Act 3: commit | protagonist's changed choice | power reversal #2; uses capability/flaw planted ≤2:15 |
| 4:05–4:20 | **Climax** (begins inside 85–93% of story runtime; 4:05 ≈ 90%) | the most expensive-feeling sequence | fight transformation beat; resolve dramatic question ON SCREEN |
| 4:20–4:30 | Resolution | earn the ending | cut on the question — 2s earlier than feels safe |
| +≤5s | Button | series signature | sting/tag only |

## 4. Fight architecture (per fight — usually 1 major + 1 minor)

Every fight is a mini-arc: **wind-up → tug-of-war (≥2 power reversals) → transformation.**

- **Location + 3–5 interactable elements** (the set is a third combatant; each environmental
  reaction gets its own shot row later):
- **Power reversal 1 (trigger prop/element):**
- **Power reversal 2 (trigger prop/element):**
- **What the loser learns / what changes in the winner:**
- Beats decompose at shotlist time into SETUP → IMPACT → REACTION molecules, one primary
  movement per 3–8s clip, contact frame never rendered.

## 5. Emotional temperature chart

One line per beat: temperature (calm/tense/violent/grief/awe…) — the chart must VARY.
All-tension reads as flat as all-calm.

## 6. Viewer knowledge ledger

Per beat: what the viewer knows before → after. Standalone test: a stranger understands stakes
by 0:15 without narration or FORP lore.

## 7. Canon anchors

Which existing FORP characters / places / motifs appear (≥1 required), and what this episode
adds to canon (new fragment, new location, new rule — each needs a vault note after release).

## 8. Runtime math

Beats sum to 4:30 ±15s before button. Attach the per-act second counts here.
